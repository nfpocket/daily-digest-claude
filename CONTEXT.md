# Daily Digest — Domain Language

## Connector
A TypeScript module in `server/connectors/` that knows how to fetch raw data from one external system (e.g. Slack, GitHub).

Implements the `Connector` interface: `{ id, name, description, fetch(config, since) }`.
Registers itself by calling `registerConnector()` at module load time.
New connectors are generated by the "Build a Connector" feature (Claude Code CLI).

## Source
A user-configured instance of a Connector with specific settings — e.g. "Slack with #engineering + DMs enabled". A Source lives inside a Schedule Entry.

A Source is not the same as a Connector: Slack is a Connector; "Slack watching #engineering" is a Source.

## Schedule Entry
A named digest job: which days of the week to run, what time, which Sources to pull from, and an optional prompt override.

Stored in `.digest/config.md` under the `schedules` array.

## Digest
The structured output Claude produces from one Schedule Entry run.
Format: `## Needs Your Attention` section at the top, then one `## [Source Name]` section per Source.
Stored as `.digest/digests/YYYY-MM-DD-HH-MM.md`.
Delivered via Slack DM + macOS desktop notification.
