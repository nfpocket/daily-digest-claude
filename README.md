# Daily Digest

A local developer tool that runs as a background service on macOS. It fetches data from connected sources (Slack, and anything else you add), summarizes everything with Claude, and delivers a structured digest on a schedule you define.

## How it works

1. **Connect** sources — Slack channels, DMs, mentions, or any data source you describe to Claude
2. **Schedule** digests — pick days and times, configure what gets included
3. **Read** — summaries land in the app and a Slack DM; the background service handles everything automatically

The app runs locally on your machine. All data stays in `.digest/` (gitignored). Claude generates the summaries using your existing Claude subscription.

## Quick start

```bash
git clone <repo>
cd daily-digest-claude
npm install
npm run dev
```

Open [http://localhost:3005](http://localhost:3005) — you'll be walked through a setup wizard.

## Setup

> **Claude auth is automatic.** Digests are generated by spawning `claude -p` as a subprocess — the same Claude Code CLI you're already logged into. No API key or extra config needed; if `claude` works in your terminal, the app works.

### 1. Slack

You need a Slack app with a **User OAuth token** (`xoxp-...`). The setup wizard gives you an app manifest to paste into [api.slack.com/apps](https://api.slack.com/apps) — create the app, install it to your workspace, copy the token.

You'll also need your **User ID** (found in your Slack profile → three-dot menu → Copy member ID).

### 3. Schedule

Name your digest, pick days (Mon–Fri by default) and a time (e.g. `07:00`). You can create multiple schedule entries with different sources.

### 4. Background service (macOS)

Install a LaunchAgent so digests run automatically at login:

```
Settings → Background Service → Install
```

This writes `~/Library/LaunchAgents/com.daily-digest.plist` and keeps the server alive across reboots.

## Connectors

A **connector** is a TypeScript module that knows how to fetch data from one external system. Each connector implements:

```typescript
interface Connector {
  id: string
  name: string
  description: string
  fetch(config: unknown, since: Date): Promise<SourceData>
}
```

`since` is the timestamp of the previous digest run (or 24 hours ago on first run), so each digest only covers what's new.

### Built-in connectors

| Connector | What it fetches |
|-----------|----------------|
| `slack` | Channel messages, DMs, mentions |

### Adding a connector via UI

**Settings → Build a Connector** — describe what you want (e.g. "GitHub issues assigned to me") and Claude will generate the connector code and wire it into the registry automatically.

### Adding a connector via CLI skill

From the project directory, run the `/add-connector` skill:

```
/add-connector Fetch my Linear issues that moved to "In Review" today
```

This calls Claude Code directly without going through the browser, which is useful when scripting or working in a terminal-only environment.

### Writing a connector manually

1. Create `server/connectors/your-name.ts`:

```typescript
import { z } from 'zod'
import type { Connector, SourceData } from './registry'

const ConfigSchema = z.object({
  // your config shape
})

export const yourConnector: Connector = {
  id: 'your-name',
  name: 'Your Name',
  description: 'What it fetches',

  async fetch(rawConfig: unknown, since: Date): Promise<SourceData> {
    const config = ConfigSchema.parse(rawConfig)
    // fetch your data, filter by since, return items
    return {
      source: 'Your Name',
      items: [{ content: '...', author: '...', timestamp: new Date(), url: '...' }],
    }
  },
}
```

2. Register it in `server/connectors/index.ts`:

```typescript
import { yourConnector } from './your-name'

const connectors: Connector[] = [
  slackConnector,
  yourConnector,   // add here
]
```

See `server/connectors/slack.ts` for a full example.

## Data storage

Everything in `.digest/` is gitignored and local to your machine:

| Path | Contents |
|------|----------|
| `.digest/config.md` | Schedule entries and source configs (YAML frontmatter) |
| `.digest/.env` | Auth tokens (`SLACK_TOKEN`, `SLACK_USER_ID`) |
| `.digest/digests/` | Generated digests — one `.md` file per run |
| `.digest/app.log` | Background service stdout |
| `.digest/app.error.log` | Background service stderr |

## Architecture

```
server/
  connectors/        # Data source integrations
    registry.ts      # Connector interface (no deps — avoids circular imports)
    index.ts         # Static connector array + getConnector / listConnectors
    slack.ts         # Slack implementation
  api/               # Nitro API routes
  utils/
    digest.ts        # Orchestrates fetch → prompt → summarize → save
    config.ts        # Read/write .digest/config.md with Zod validation
    claude.ts        # Spawns `claude -p` subprocess (inherits your CLI login)
    launchagent.ts   # Installs/uninstalls macOS plist
    notify.ts        # Desktop notification + Slack DM
  plugins/
    env.ts           # Loads .digest/.env into process.env at startup
    scheduler.ts     # node-cron: fires digest runs on schedule
app/
  pages/
    index.vue        # Digest viewer (history sidebar + content)
    settings/        # Schedule + connector management
    setup/           # Onboarding wizard
  composables/
    useDigestRun.ts  # EventSource for real-time digest progress
```

Digest flow: `scheduler.ts` → `digest.ts` → `connector.fetch()` per source → build prompt → `claude -p` → save `.md` → notify.
